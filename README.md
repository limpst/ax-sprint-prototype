## 1. 이 프로그램이 무엇인가요?

SH공사·경상북도개발공사 관리 공공임대주택의 **안전·행정을 AI와 RPA가 자동으로 처리**하는 웹 대시보드입니다.

**한 줄 요약**
> 드론·앱으로 건물 손상을 자동 탐지하고, IoT·AI가 사고 발생 전 선제 차단하며, 민원·관리비·계약은 RPA 로봇이 자동 처리

**3대 핵심 축**
```

│                           에이톰-AX 플랫폼                              │
│                                                                       │
│  ① 비전 AI 정밀진단          ② AI 사전 예방 정비          ③ RPA 행정 자동화  │
│  ─────────────────          ─────────────────            ────────────── │
│  · 균열 0.2mm 자동 탐지      · IoT 이상 징후 조기 감지     · 관리비 고지 80%  │
│  · 드론 + 입주민 앱 이중 수집 · LiDAR 3D Digital Twin       · 계약갱신 알림 100%│

│  · 손상 발생 前 경보 자동 발송 · 예방 정비 우선순위 자동 결정  · 연간 1억원 절감  │
```

```python
노후도점수 = max(20, 95 - 경과년수 × 1.8 + uniform(-5, 5))

# 강남 A동 (1998년, 28년 경과): max(20, 95 - 50.4) = 44.6점 → 저위험 (빨간 막대)
# 강서 D동 (2018년, 8년 경과):  max(20, 95 - 14.4) = 80.6점 → 고위험 (초록 막대)
```

### 10-1. AI 기반 GPR 데이터 자동 해석 및 성장 예측

#### 개요

**지표투과레이더(GPR, Ground Penetrating Radar)** 탐사 데이터를 딥러닝으로 분석하는 것이 가장 직접적인 지반침하 예방 기술입니다. 기존 전문가 수작업 해석 대비 분석 시간을 **90% 이상** 단축하고, 주관적 판단에 의존하던 신뢰도를 AI 기반 정량화된 수치로 대체합니다.

---

**활성 공동 판별 기준:**

```python
# 공동 활성도 판별 로직 (의사코드)
def classify_void_activity(growth_rates: list[float]) -> str:
    """
    growth_rate: 분기별 면적 증가율 (cm²/월)
    """
    recent_rate   = growth_rates[-1]        # 최근 1개월 성장률
    trend_slope   = np.polyfit(range(len(growth_rates)), growth_rates, 1)[0]

    if recent_rate > 15.0 or trend_slope > 3.0:
        return "CRITICAL"   # 즉시 굴착 보수 (24h 내)
    elif recent_rate > 5.0 or trend_slope > 1.0:
        return "ACTIVE"     # 2주 내 긴급 조치
    elif recent_rate > 1.0:
        return "GROWING"    # 월 1회 집중 모니터링
    else:
        return "STABLE"     # 분기 1회 정기 점검
```

#### C. 유지보수 우선순위 결정 알고리즘

공동의 **크기, 성장 속도, 위치 위험도** 3개 축을 결합한 복합 위험도 스코어로 보수 순서를 자동 결정합니다.

```
유지보수 우선순위 산정 매트릭스

  위험도 스코어 (Risk Score) = α·S + β·G + γ·L

  S (크기 점수, 0~40):
    · 직경 < 30cm     → 10점
    · 30cm ≤ 직경 < 60cm → 20점
    · 60cm ≤ 직경 < 100cm → 30점
    · 직경 ≥ 100cm    → 40점

  G (성장 점수, 0~35):
    · STABLE          → 5점
    · GROWING         → 15점
    · ACTIVE          → 25점
    · CRITICAL        → 35점

  L (위치 위험 점수, 0~25):
    · 보도/녹지       → 5점
    · 이면도로        → 10점
    · 차도(편도 1차로) → 15점
    · 간선도로(2차로+) → 20점
    · 공공주택 건물 직하 → 25점

  α = 1.0,  β = 1.2,  γ = 1.5  (가중치, 현장 데이터로 보정)

  ┌────────────────┬──────────────┬────────────────────────────┐
  │  최종 스코어   │  위험 등급   │  조치 기준                  │
  ├────────────────┼──────────────┼────────────────────────────┤
  │  80 ~ 100      │  🔴 CRITICAL │  24시간 내 긴급 굴착 보수   │
  │  60 ~  79      │  🟠 HIGH     │  1주일 내 계획 보수         │
  │  40 ~  59      │  🟡 MEDIUM   │  1개월 내 예방 보수         │
  │  20 ~  39      │  🔵 LOW      │  분기별 집중 관찰           │
  │   0 ~  19      │  🟢 SAFE     │  정기 탐사 주기 유지        │
  └────────────────┴──────────────┴────────────────────────────┘
```

## 13. 자주 묻는 질문

**Q. 실제 AI 모델이 탑재되어 있나요?**
> A. 프로토타입입니다. Y-MaskNet 대신 PIL 이미지 생성으로 시뮬레이션합니다. 실제 모델은 세종대학교와 공동 개발 중이며 v2.0에서 연동됩니다.

**Q. 데이터가 매번 바뀌나요?**
> A. `np.random.seed(2026)`, `random.seed(2026)`으로 시드를 고정했기 때문에 **항상 동일한 데이터**가 생성됩니다. 단 일부 차트(월별 추이 등)는 렌더링마다 랜덤 재생성됩니다.

**Q. 브라우저를 새로고침하면 데이터가 사라지나요?**
> A. 새로고침은 `st.session_state`가 유지되어 데이터가 보존됩니다. 탭을 완전히 닫고 다시 열면 초기화됩니다.

**Q. 인터넷 없이 실행되나요?**
> A. 라이브러리 설치 후 완전 오프라인 실행 가능합니다.

**Q. 스마트폰에서도 볼 수 있나요?**
> A. 같은 네트워크에서 `http://[컴퓨터IP]:8501`로 접속 가능합니다.

**Q. GPR 탐사 데이터는 어떤 파일 형식을 지원하나요?**
> A. v2.0에서는 **SEG-Y 형식** (산업 표준)과 각 GPR 장비 제조사별 독점 형식(Mala .rd3, GSSI .dzt, IDS .dt, Leica .sgr)을 Python 라이브러리(ReadSegy, GPRPy)를 통해 파싱 예정입니다. 현재 프로토타입은 CSV 기반 샘플 데이터를 사용합니다.

**Q. InSAR 데이터는 유료인가요?**
> A. **Sentinel-1 위성 데이터는 무료**입니다 (유럽우주국 Copernicus 프로그램). ALOS-2(JAXA, 유료) 등 고해상도 위성은 필요시 상업 구매가 필요합니다. 서울시 일대는 Sentinel-1이 6~12일 주기로 무료 제공하므로 초기 운영에는 무료 데이터만으로도 충분합니다.

**Q. DAS 광섬유 설치 비용이 너무 크지 않나요?**
> A. 신규 통신 관로 공사 시 **기존 매설 광섬유를 DAS 센서로 전용(轉用)** 하는 방식을 채택하면 추가 설치 비용을 최소화할 수 있습니다. 공동주택 단지 내 기 매설 광섬유 통신선이 있다면 인터로게이터(약 5,000~1억원 장비) 추가만으로 즉시 적용 가능합니다.

**Q. 지반침하 예방 모듈은 언제부터 사용 가능한가요?**
> A. v1.1.0(2026년 5월 예정)에서 지반침하 신호등 UI 프로토타입이, v2.0.0(2026년 하반기)에서 GPR AI 엔진과 DAS 연동이, v3.0.0(2027년)에서 STL-XGBoost 예측 모델과 AI 위험 지도가 순차 탑재될 계획입니다.

**Q. 진동-침하 상관관계 분석은 어떤 데이터가 필요한가요?**
> A. 최소 **1년 이상의 시계열 데이터** 가 필요합니다: IoT 진동 가속도 센서 (15분 간격), InSAR 지표 침하량 (월 1~2회), 경사계 기울기 (15분 간격), 강수량·지하수위 데이터. 이태리하우스 사례처럼 공사 진동이 잦은 단지 2~3개소를 우선 집중 계측하면 1~2년 내 유의미한 상관계수 도출이 가능합니다.

## 14. Render 배포 가이드 — 아주 쉬운 단계별 설명

> **Vercel은 Streamlit과 호환되지 않습니다.** Vercel은 정적 사이트 / 짧은 함수 전용이라 Streamlit처럼 계속 실행되는 서버 앱은 동작하지 않습니다.
> **Render** 를 사용하세요 — 무료 플랜으로 Streamlit을 완전히 지원합니다.

```
✅ GitHub 계정          — github.com 에서 무료 가입
✅ Render 계정          — render.com 에서 GitHub 로그인으로 가입 (무료)
✅ 이 프로젝트 파일들    — render.yaml, .streamlit/config.toml 이미 포함되어 있음
```

### STEP 1 — GitHub에 코드 올리기 (최초 1회)

#### 1-1. GitHub에 새 저장소(Repository) 만들기

```
1. github.com 접속 → 로그인
2. 오른쪽 상단 + 버튼 → "New repository" 클릭
3. Repository name: atomax-platform  (이름은 자유)
4. Public 선택 (무료 배포에 필요)
5. "Create repository" 버튼 클릭
```

#### 1-2. 코드 GitHub에 업로드

터미널(cmd)에서:

```bash
# 프로젝트 폴더로 이동
cd C:\Users\on\PycharmProjects\PythonProject3

# GitHub 저장소 연결 (yourname 을 본인 GitHub 아이디로 변경)

# 전체 파일 업로드
git add .
git commit -m "deploy: Render 배포 설정 추가"
git push origin main
```

> **주의:** `.env` 파일은 `.gitignore` 에 이미 포함되어 자동으로 제외됩니다. 비밀키가 올라가지 않으니 안심하세요.

### STEP 2 — Render에서 서비스 만들기

### STEP 3 — render.yaml 로 자동 설정하는 방법 (더 쉬운 방법)

```
1. render.com → 대시보드 → "New +" → "Blueprint"

3. render.yaml 파일이 자동으로 감지됨

```

### STEP 4 — 코드 수정 후 재배포 (자동)

`render.yaml` 에 `autoDeploy: true` 가 설정되어 있어서
**GitHub에 push하면 Render가 자동으로 새 버전을 배포합니다.**

```bash
# 코드 수정 후
git add .
git commit -m "update: 기능 개선"
git push origin main
# → Render가 자동으로 감지하고 재빌드·재배포
```

### 무료 플랜 제한 사항

| 항목 | 무료 플랜 |
|------|-----------|
| 월 실행 시간 | 750시간 (사실상 항상 켜짐) |
| 15분 비활성 후 | 슬립 모드 진입 (첫 접속 시 30초~1분 딜레이) |
| 메모리 | 512MB |
| CPU | 공유 0.1 vCPU |
| 커스텀 도메인 | 가능 (무료) |
| HTTPS | 자동 발급 (무료) |

> **슬립 모드 해결:** 유료 플랜($7/월)으로 업그레이드하거나, UptimeRobot 같은 무료 서비스로 15분마다 ping을 보내면 슬립 없이 유지됩니다.

### 배포 파일 구조 설명

```
프로젝트/
│
├── render.yaml              ← Render 배포 설정 (빌드·시작 명령어 자동화)
├── .streamlit/
│   └── config.toml          ← Streamlit 서버 설정 (포트, 테마, 보안)
├── requirements.txt         ← 설치할 패키지 목록 (버전 고정)
├── app.py                   ← 실제 앱 코드 (이걸 Render가 실행)
└── main.py                  ← 로컬 실행 전용 (Render에서는 미사용)
```

### 문제 해결

| 증상 | 원인 | 해결 방법 |
|------|------|-----------|
| 빌드 실패 "ModuleNotFoundError" | requirements.txt 패키지 누락 | 해당 패키지 requirements.txt에 추가 후 재push |
| 앱은 뜨는데 화면이 안 나옴 | 포트 문제 | Start Command에 `--server.port $PORT` 있는지 확인 |
| 15분 후 접속 안 됨 | 슬립 모드 | 재접속 시 30초 대기 (정상), 또는 UptimeRobot 설정 |
| 한글 깨짐 | Python 인코딩 | `app.py` 상단에 `import sys; sys.stdout.reconfigure(encoding='utf-8')` 추가 |
| 빌드는 성공인데 에러 화면 | app.py 런타임 오류 | Render 대시보드 → 서비스 → Logs 탭에서 에러 확인 |
| GitHub push해도 재배포 안 됨 | autoDeploy 꺼짐 | Render 대시보드 → Settings → Auto-Deploy 활성화 |
