## 1. 이 프로그램이 무엇인가요?

이 프로그램은 **공공주택(SH공사 등)** 의 건물 안전을 AI가 자동으로 관리해 주는 **웹 대시보드** 입니다.

쉽게 말하면:

> "건물에 금이 갔는지, 물이 새는지, 화재 위험은 없는지 — 사람이 일일이 돌아다니며 확인하는 것을 AI와 센서가 대신 해주는 프로그램"

**국토교통부 AX-SPRINT(AI 응용제품 신속 상용화 지원사업)** 과제의 프로토타입으로,
실제 SH공사(서울주택도시공사)와 협력하여 현장 실증을 목표로 개발 중입니다.

## 2. 왜 이 플랫폼이 필요한가요?

### 현재의 문제점

| 문제 | 현재 상황 | 이 플랫폼 적용 후 |
|------|-----------|-----------------|
| 건물 점검 | 사람이 직접 육안으로 확인 → 놓치는 것 많음 | AI + 드론이 0.2mm 균열까지 자동 탐지 |
| 민원 처리 | 평균 **8개월** 걸림 (분쟁, 서류 왔다갔다) | **4일 이내** 자동 분석·처리 |
| 책임 판단 | "공사 책임이냐 입주민 책임이냐" 다툼 | AI가 객관적 증거로 즉시 판정 |
| 보험 분쟁 | 손해액 산정 기준 없어 민원 반복 | XAI 보고서로 손해액 자동 산출 |
| 예산 낭비 | 사고 난 후에야 수리 → 비용 2~3배 | 사전 예방으로 유지보수 비용 20% 절감 |

### 해결 방향

```
[기존]  사고 발생 → 민원 접수 → 현장 점검 → 책임 공방 → 수리 (평균 8개월)
[신규]  AI 사전 감지 → 자동 분석 → 즉시 조치 → 객관적 보고서 발급 (4일)
```

## 3. 주요 기능 5가지

### 기능 1: 📊 통합 대시보드
- **무엇을 보여주나요?**
  - 전체 관리 세대 수, 점검 완료율, 고위험 민원 건수, AI 정확도, 평균 처리 기간을 한 눈에 확인
  - 실시간 위험 알림 (빨간색 = 즉시조치, 노란색 = 주의, 초록색 = 정상)
  - 월별 손상 탐지 추이 그래프
  - 단지별 노후도 점수 순위
- **비전공자 설명:** 병원의 환자 상태판처럼, 모든 건물의 상태를 한 화면에 요약해서 보여주는 메인 화면입니다.

### 기능 2: 🔍 AI 손상 탐지
- **무엇을 하나요?**
  - 드론이 촬영한 건물 외벽 사진, 또는 입주민이 앱으로 찍은 사진을 업로드하면
  - AI(Y-MaskNet / Mask R-CNN)가 균열·누수·박리 등을 자동으로 찾아냄
  - 균열 폭이 0.2mm 이상이면 자동으로 "고위험" 분류 후 담당자에게 알림
  - 탐지 후 작업지시서(Work Order)를 자동 생성
- **비전공자 설명:** 건물 사진을 AI에게 보여주면 "여기 금 갔어요", "저기 물 샌 흔적 있어요"라고 알려주는 기능입니다.

### 기능 3: 📡 IoT 실시간 모니터링
- **무엇을 하나요?**
  - 건물 곳곳에 설치된 센서가 **온도, 습도, 진동, CO(일산화탄소) 농도** 를 실시간으로 측정
  - 기준치 초과 시 자동으로 담당자에게 알림 발송
  - 최근 1~30일 데이터를 그래프로 시각화
  - 이상 감지 이벤트 로그 자동 기록
- **비전공자 설명:** 건물에 온도계, 습도계, 지진감지기, 가스감지기를 달아두고 24시간 자동으로 체크하는 기능입니다.

### 기능 4: 📋 민원 관리 (RPA 자동화)
- **무엇을 하나요?**
  - 입주민이 앱으로 민원 접수 → AI가 자동으로 위험도 분류(고/중/저)
  - 고위험이면 즉시 담당자에게 자동 알림
  - 민원 유형별·상태별 필터링 및 목록 관리
  - 월별 민원 통계 및 처리 기간 분포 차트
- **비전공자 설명:** 입주민이 "화장실 천장에서 물이 새요"라고 신고하면, AI가 자동으로 얼마나 위험한지 판단하고 담당자에게 알려주는 기능입니다.

### 기능 5: 📄 XAI 보고서 자동 생성
- **무엇을 하나요?**
  - 사고 발생 시 AI가 "공사 책임인지, 입주민 책임인지" 자동으로 판정
  - KICT(한국건설기술연구원) 가이드라인 기준 적용
  - 판정 근거를 그래프로 설명 (어떤 요소가 얼마나 영향을 미쳤는지)
  - 손해액을 자동 산출하여 PDF 보고서 생성
- **비전공자 설명:** 법원 판사처럼, AI가 증거(사진, 센서 데이터, 이력)를 보고 "이건 건물이 낡아서 생긴 거니 공사 책임"이라고 객관적으로 판정해주는 기능입니다.

### 7-2. AI 손상 탐지 로직

```python
# 핵심 판단 기준 (KICT 가이드라인 기반)
균열 폭 기준:
  0.2mm 미만  → "저위험"  (정기점검 유지)
  0.2~0.5mm  → "중위험"  (30일 내 재점검)
  0.5mm 이상  → "고위험"  (즉시 조치 필요)

# AI 모델 구조 (실제 구현 시)
Y-MaskNet (에이톰 자체 개발)
  └── Mask R-CNN 기반 인스턴스 세그멘테이션
       └── 균열 픽셀 단위 분리 → 실제 크기 변환 (드론 고도 기반 스케일링)
```

**프로토타입에서의 구현:**
- PIL 라이브러리로 가상의 균열 이미지를 생성하여 시각화
- 실제 서비스에서는 학습된 딥러닝 모델로 교체

### 7-3. IoT 센서 임계값 로직

```python
# 각 센서별 경고 기준
온도    > 30°C   → 경고 (열화 위험)
습도    > 70%    → 경고 (누수·곰팡이 위험)
진동    > 0.5g   → 경고 (구조 이상 의심)
CO 농도 > 10ppm  → 경고 (화재·가스 위험)

# 경고 발생 시 자동 동작
1. 대시보드 알림 표시 (빨간색)
2. 담당자 문자/이메일 자동 발송 (실제 구현 시 연동)
3. 이벤트 로그 자동 기록
```

---

### 7-4. 민원 처리 자동화 로직 (RPA)

```
[민원 접수]
  입주민 앱 → 사진 + 텍스트 입력

[AI 자동 분류] (1~2초 내)
  유형 분류: 누수 / 균열 / 화재위험 / 시설노후 / 기타
  위험도 배정:
    화재위험, 균열 → 자동으로 "고위험"
    누수           → "중위험"
    시설노후, 기타  → "저위험"

[자동 후속 조치]
  고위험 → 담당팀장에게 즉시 알림 + 긴급 작업지시
  중위험 → 담당자에게 알림 + 3일 내 처리 큐 등록
  저위험 → 정기점검 큐 등록

[기존 프로세스 대비]
  접수 → 담당자 배정: 3일 → 즉시 (자동)
  현장 점검 예약:     7일 → 자동 스케줄링
  책임 판정:         수개월 → AI가 즉시
  전체 처리:         평균 8개월 → 4일
```

### 7-5. XAI 보고서 로직 (설명 가능 AI)

```
[XAI = eXplainable AI = 설명 가능한 AI]

일반 AI: "이건 공사 책임입니다" → 왜? 모름
XAI:    "이건 공사 책임입니다"
         왜냐하면:
           - 건물이 22년 됐고 (기여도 28%)
           - 최근 1년에 같은 세대 누수 3번 (기여도 23%)
           - 습도 센서 70% 초과 8회 감지 (기여도 18%)
           - AI 탐지 손상면적 45cm² (기여도 20%)
           - 기타 (11%)
         → 이런 근거로 판정했습니다 (법원에서도 인정 가능)

# SHAP 값 기반 기여도 계산
factors = {
    "건물 준공 연도 (노후도)":        0.28,
    "최근 점검 이력":                0.23,
    "IoT 센서 이상 감지 횟수":        0.18,
    "AI 균열·손상 탐지 면적":         0.20,
    "입주민 신고 이력":               0.07,

}
```

## 9. 샘플 데이터 설명

프로토타입은 실제 데이터 없이도 바로 실행됩니다. 모든 데이터는 코드가 자동으로 만들어냅니다.

### IoT 센서 데이터 (`make_sensor_timeseries`)

```
30일치 × 24시간 = 720개 데이터 포인트 자동 생성

온도: 22°C 기준으로 하루 주기로 오르내림 + 약간의 랜덤 변동
      (새벽에 낮고, 낮에 높은 패턴 - 현실적)

습도: 55% 기준으로 파형 생성 + 랜덤 변동

진동: 대부분 0에 가깝지만, 특정 기간(200~220시간)에 0.8g 이상 이상값 삽입
      → 이것이 "이상 감지 이벤트"로 로그에 기록됨

CO 농도: 대부분 2ppm이지만 400~410시간에 15ppm 급등
          → 경고 이벤트 발생 시뮬레이션
```

### 민원 데이터 (`make_complaints`)

```
60건의 민원 자동 생성:
- 5개 단지 중 랜덤 배정
- 접수일: 최근 90일 내 랜덤
- 상태: 접수(15%) / AI분석중(20%) / 처리중(30%) / 완료(35%) 비율로 배정
- AI점수: 0.60~0.99 사이 랜덤 (AI 신뢰도)
```

### 점검 이력 데이터 (`make_inspection_data`)

```
5개 단지 × 12개월 = 60개 레코드

각 월별로:
- 균열건수: 0~8건 (랜덤)
- 누수건수: 0~5건 (랜덤)
- 화재위험건수: 0~3건 (랜덤)
- 노후도점수: 40~85점 (낮을수록 노후, 높을수록 양호)
```

### AI 손상 탐지 결과 (`make_damage_detection_result`)

```
5개 실제 탐지 결과 시나리오:
1. 외벽 3층 북측 - 수직균열 0.35mm - 고위험 - AI신뢰도 94%
2. 지하 1층 천장 - 누수흔적 0.12mm - 중위험 - AI신뢰도 88%
```

## 10. 자주 묻는 질문 (FAQ)

**Q. 실제 AI 모델이 탑재되어 있나요?**
> A. 이 버전은 프로토타입으로, 실제 AI 모델 대신 시뮬레이션 데이터를 사용합니다. 실제 Y-MaskNet 모델은 세종대학교와 함께 개발 중이며, 2차년도에 연동됩니다.

**Q. 실제 SH공사 데이터를 쓸 수 있나요?**
> A. 현재는 샘플 데이터만 사용합니다. 과제 선정 후 SH공사 협약에 따라 비식별화된 실제 데이터를 연동할 예정입니다.

**Q. 인터넷이 없어도 실행되나요?**
> A. 네! 라이브러리만 설치되어 있으면 인터넷 없이 로컬에서 실행됩니다.

**Q. 모바일에서도 볼 수 있나요?**
> A. Streamlit은 반응형 웹이라 스마트폰 브라우저에서도 접속 가능합니다. 같은 네트워크라면 `http://[내컴퓨터IP]:8501` 로 접속하세요.

**Q. 데이터를 저장할 수 있나요?**
> A. 현재는 새로고침 시 초기화됩니다. 실제 서비스에는 데이터베이스 연동이 추가됩니다.
