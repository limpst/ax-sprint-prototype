## 1. 이 프로그램이 무엇인가요?

이 프로그램은 **SH공사(서울주택도시공사)** 와 **경상북도개발공사** 가 관리하는
공공임대주택의 **안전을 AI가 자동으로 관리**해주는 **웹 대시보드 플랫폼**입니다.

### 한 줄 요약

> "건물의 균열·누수·화재위험을 드론과 AI가 자동 발견하고,
> 민원·관리비·계약 업무는 RPA 로봇이 자동 처리하며,
> 분쟁이 생기면 AI가 객관적 증거로 즉시 판정하는 통합 플랫폼"

### 이 플랫폼의 3대 축

```
[1] 비전 AI 정밀진단        →  균열 0.2mm까지 자동 탐지 (드론 + 앱)
[2] RPA 지능형 행정자동화   →  관리비·계약·민원·점검 자동 처리
[3] XAI 손해사정 보고서     →  분쟁 8개월 → 즉시 종결 (KICT 기준)
```

### ✈ 기능 2: 비전AI · 드론 정밀진단

**핵심 기술:**
- **Y-MaskNet** (에이톰 자체 개발): 균열·누수·드라이비트 탐지 특화 모델
- **Mask R-CNN**: 인스턴스 세그멘테이션 — 손상 부위를 픽셀 단위로 분리
- 탐지 가능 항목:
  - 수직·수평 균열 (폭 **0.2mm ~ 2.0mm** 정량 측정)
  - 누수 흔적 (습윤면적)
  - **드라이비트(가연성 외벽재) 화재취약 구조** (KFS 기준)
  - 박리·박락·표면손상

**입력 방식 2가지:**
1. **입주민 앱 업로드**: 세대 내부(화구, 수전 등) 촬영 → AI 실시간 판별
2. **HSIA 드론 스캔**: 외벽·옥상·공용부 항공 촬영 → 자동 분석

**출력:**
- AI 세그멘테이션 오버레이 이미지
- 손상 유형·폭·위험도·AI신뢰도 자동 산출
- **작업지시서(Work Order)** 자동 생성

**판정 기준 (KICT + KFS):**
```
균열 폭  < 0.2mm  →  저위험 (정기점검 유지)
균열 폭 0.2~0.5mm →  중위험 (30일내 재점검)
균열 폭 > 0.5mm   →  고위험 (즉시 구조전문가 요청)
드라이비트 외벽   →  고위험 (KFS 화재안전기준 적용)
```

**세움터·FMS 연동:**
- 세움터(건축물 도면) + FMS(시설물정보) + 드론 스캔 → **3D 디지털 트윈** 구축
- 건물 전체 수명주기 노후도 자동 예측

### 🤖 기능 3: RPA 지능형 행정 자동화

**RPA = Robotic Process Automation = 로봇 프로세스 자동화**
(사람이 컴퓨터로 반복하던 작업을 프로그램이 대신하는 기술)

**자동화 업무 및 달성률 (문서 기준):**

| 업무 | 자동화율 | 처리 변화 |
|------|----------|-----------|
| 관리비 고지서 발행 | **80%** | 4시간 → 48분/월 |
| 계약 만료 알림 발송 | **100%** | 수동 → 완전 자동 |
| 민원 접수·분류 | **70%** | 30분 → 9분/건 |
| 점검 일정 자동 생성 | **90%** | 2시간 → 12분/회 |
| 담당자 자동 배정 | **75%** | 20분 → 5분/건 |
| 점검 결과 보고서 | **90%** | 3시간 → 자동 생성 |

**연간 절감 효과: 약 1,350만원 (목표: 1억원)**

**세부 기능:**
- **관리비 고지서**: 단지별 세대별 자동 생성·발행, 오류 세대 자동 표시
- **계약 만료 알림**: 60일/30일/7일 전 자동 문자 발송 (RPA 100%)
- **AI 점검 일정**: 노후도점수 × 마지막점검일 × 손상이력 → 최적 일정 자동 수립

### 📋 기능 4: AI 민원 관리

**민원 처리 워크플로우 (6단계 완전 자동화):**

```
[1단계] 민원 접수      입주민 앱 사진+텍스트 → 자동 접수번호 발급
[2단계] AI 자동 분류   유형·위험도 자동 분류 (AI 신뢰도 72~97%)
[3단계] 담당자 자동배정 위험도·단지·업무량 기반 최적 담당자 자정
[4단계] 현장 조치      드론/앱 재점검 → 실측 → 보수 시행
[5단계] XAI 손해 산정  KICT 기준 책임소재 판정 + 손해액 자동 산출
[6단계] 분쟁 종결      객관적 AI 보고서 발급 → 입주민 확인·서명
```

**민원 유형별 위험도 자동 배정:**
- 균열·구조손상, 화재·가연성외벽 → **고위험** (즉시 담당팀장 알림)
- 누수 → **중위험** (3일내 현장점검)
- 시설노후, 관리비이의, 계약관련, 기타 → **저위험** (정기점검 큐)

### 📄 기능 6: XAI 손해사정 보고서

**XAI = eXplainable AI = 설명 가능한 AI**

```

에이톰 XAI: "공사 책임입니다"
  근거 1: 건물 경과 22년 (기여도 28%)  ← 노후도 기준 초과

  근거 4: AI 손상 탐지 면적 45cm² (20%)  ← 계측값
  근거 5: 기타 (11%)
  ↓
  → 법원에서도 인정 가능한 객관적 증거 보고서
  → 분쟁 종결: 8개월 → 즉시~2주
```

**KICT 기준 적용:**

- 책임소재 판별 표준 AI 학습

**출력물:**

- 손해액 자동 산출 (입주민 청구액 대비 AI 인정액)

### 6-2. 샘플 데이터 생성 함수 (코드 레벨 설명)

`app.py` 안에는 **데이터를 자동으로 만드는 함수**들이 있습니다.
실제 DB 없이도 바로 실행되는 이유입니다.

```python
# [함수 1] IoT 센서 데이터 생성
make_iot_data(days=30)
  → 30일 × 24시간 = 720개 데이터 포인트
  → 온도: 21°C 기준 sin파형 (낮에 높고 새벽에 낮은 현실적 패턴)
  → 습도: 54% 기준 sin파형
  → 진동: 대부분 정상이지만 특정 기간 0.85g 이상값 삽입 (이상 이벤트 시뮬레이션)
  → CO:   대부분 1.8ppm이지만 특정 기간 16ppm 급등 (경보 시뮬레이션)

# [함수 2] 민원 데이터 생성
make_complaints(n=80)
  → 80건의 민원 자동 생성
  → 유형별 위험도 자동 배정 (균열/화재=고, 누수=중, 기타=저)
  → RPA 자동화율: 유형별로 70~100% 설정
  → 상태: 접수(8%) / AI분류완료(12%) / 담당자배정(15%) / 처리중(25%) / 완료(30%) / 분쟁종결(10%)

# [함수 3] 시설물 대장 생성
make_facilities()
  → 단지별 준공년도 → 경과년수 자동 계산
  → 노후도점수: 95 - 경과년수 × 1.8 (오래될수록 낮은 점수)
  → 외벽재질: 드라이비트 / 타일 / 노출콘크리트 / 알루미늄복합판넬
  → FMS/세움터 연동 상태 포함

# [함수 4] AI 손상 탐지 결과
make_damage_detections()
  → 8개의 실제 탐지 시나리오 (문서 기반)
  → 균열폭 / 손상길이 / 위험도 / AI신뢰도 / 탐지방법 / 판정기준 포함

# [함수 5] 고장 예측 데이터
make_failure_forecast()
  → 향후 12개월 예측
  → AI 적용 전: 기준값 × 1.3 (더 많은 고장)
  → AI 적용 후: 최적 예측값
  → 신뢰구간 95% 자동 생성
```

## 11. 자주 묻는 질문 (FAQ)

**Q. 실제 AI 모델이 탑재되어 있나요?**
> A. 이 버전은 **프로토타입**입니다. 실제 Y-MaskNet 딥러닝 모델 대신 시뮬레이션 데이터와
> PIL 이미지 생성을 사용합니다. 실제 모델은 세종대학교와 공동 개발 중이며 2차년도에 연동됩니다.

**Q. SH공사 실제 데이터를 사용하나요?**
> A. 현재는 샘플 데이터만 사용합니다. 과제 선정 후 SH공사 현장실증 확약서에 따라
> 비식별화된 실제 데이터(사고이력, 세움터 도면, FMS)를 연동합니다.

**Q. 인터넷이 없어도 실행되나요?**
> A. 네, 라이브러리만 설치되면 **완전 오프라인**으로 실행됩니다.

**Q. 스마트폰에서도 볼 수 있나요?**
> A. Streamlit은 반응형 웹이라 같은 네트워크에서 `http://[컴퓨터IP]:8501`로 접속 가능합니다.

**Q. 새로고침하면 입력한 데이터가 사라지나요?**
> A. 새로고침은 괜찮습니다. 다른 페이지로 이동해도 데이터가 유지됩니다.
> 다만 브라우저를 완전히 닫고 다시 열면 초기화됩니다.

**Q. 데이터를 저장하거나 내보낼 수 있나요?**
> A. XAI 보고서는 `.txt`로 다운로드 가능합니다. 실제 서비스에서는 DB 연동 및 PDF 내보내기가 추가됩니다.
